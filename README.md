# Система анализа текстовых диалогов

## Описание проекта

**Система анализа текстовых диалогов** — это приложение для автоматической обработки клиентских обращений. Проект позволяет загружать CSV-файл с диалогами, определять тему обращения, анализировать эмоциональную окраску, выявлять проблемные сценарии и находить похожие обращения.

Приложение разработано на Python с использованием Streamlit и предназначено для демонстрации методов обработки естественного языка в задачах анализа пользовательских сообщений.

## Основные возможности

- загрузка диалогов из CSV-файла;
- классификация обращений по темам;
- эмоциональный анализ сообщений;
- выявление проблемных обращений;
- поиск похожих обращений;
- отображение статистики по темам и эмоциям;
- вывод проблемных сценариев в отдельной вкладке;
- экспорт результатов анализа в CSV и Excel.

## Темы обращений

Система определяет следующие темы:

- доставка;
- качество товара;
- оплата;
- аккаунт;
- возврат;
- благодарность.

## Эмоциональный анализ

Для каждого диалога определяется эмоциональная окраска:

- позитивный;
- нейтральный;
- негативный.

Дополнительно используются правила постобработки, которые помогают точнее определять сильный негатив или позитив по ключевым словам.

## Проблемные сценарии

Диалог считается проблемным, если в нём встречаются признаки жалобы или затруднения клиента, например:

- заказ не пришёл;
- товар сломался;
- обнаружен брак;
- ошибка оплаты;
- деньги списали дважды;
- курьер опоздал;
- клиент просит вернуть деньги.

## Поиск похожих обращений

Поиск похожих обращений работает на основе сравнения текста запроса с загруженными диалогами. Пользователь вводит текст обращения, после чего система выводит наиболее похожие диалоги с процентом совпадения.


## Формат входного CSV-файла

Файл должен содержать колонку `dialog`.

Пример:

```csv
dialog
"Клиент: Заказ не пришёл. Оператор: Проверим. Клиент: Спасибо."
"Клиент: Товар сломался. Оператор: Вернём деньги. Клиент: Хорошо."
"Клиент: Курьер опоздал. Оператор: Извините. Клиент: Недопустимо."
```

## Установка зависимостей

Рекомендуется создать виртуальное окружение:

```bash
python -m venv .venv
```

Активация виртуального окружения в Windows:

```bash
.venv\Scripts\activate
```

Установка библиотек:

```bash
pip install streamlit pandas sentence-transformers chromadb transformers torch scikit-learn accelerate openpyxl
```

## Запуск приложения

Для запуска Streamlit-приложения выполните команду:

```bash
streamlit run app.py
```

После запуска откроется веб-интерфейс, в котором можно загрузить CSV-файл и выполнить анализ диалогов.

## Работа с приложением

1. Откройте приложение через браузер.
2. В боковой панели загрузите CSV-файл с колонкой `dialog`.
3. Нажмите кнопку **Анализировать**.
4. После обработки появятся метрики:
   - всего диалогов;
   - количество проблемных обращений;
   - количество негативных обращений;
   - количество позитивных обращений.
5. Во вкладке **Диалоги** можно посмотреть все обработанные записи.
6. Во вкладке **Проблемные** отображаются обращения, в которых найдены признаки проблемного сценария.
7. Во вкладке **Статистика** отображаются графики распределения по темам и эмоциям.
8. Во вкладке **Поиск** можно найти похожие обращения по введённому тексту.
9. Результаты анализа можно скачать в формате CSV или Excel.

## Генерация данных для обучения

Перед обучением моделей необходимо сгенерировать обучающие CSV-файлы для классификации тем и эмоционального анализа.

Для генерации данных по темам выполните команду:

```bash
python generate_topic_data.py
```

Для генерации данных по эмоциям выполните команду:

```bash
python generate_emotion_data.py
```

После выполнения команд будут созданы файлы:

```text
training_data_topics.csv
training_data_emotions.csv
```

## Обучение моделей

Для обучения модели классификации тем используется файл:

```bash
python train_topic_model.py
```

Для обучения модели эмоционального анализа используется файл:

```bash
python train_emotion_model.py
```

После обучения модели сохраняются в папки:

```text
topic_model/
emotion_model/
```

Основное приложение автоматически загружает эти модели при запуске, если папки существуют.

## Назначение проекта
